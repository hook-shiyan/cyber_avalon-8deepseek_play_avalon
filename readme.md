```
cyber_avalon/
│
├── main.py                  # 程序的总入口，负责初始化总线、加载 AI 并启动事件循环
├── config.py                # 全局配置文件（LLM API 密钥、模型名称、最大休眠时间等）
│
├── core/                    # 核心机制层
│   ├── message_bus.py       # 聊天室/消息总线（负责接收消息、广播信号、记录历史记录）
│   └── game_engine.py       # 阿瓦隆规则状态机（管理轮次、判定胜负、分发身份、发起投票）
│
├── agents/                  # 智能体层
│   ├── base_agent.py        # AI 玩家的基类（封装调用 API 的底层逻辑、JSON 解析、重试机制）
│   └── avalon_agent.py      # 继承自基类，专门处理阿瓦隆逻辑（评估发言意愿、执行休眠打断、生成对话）
│
└── utils/                   # 工具层
    ├── prompts.py           # 集中存放所有繁杂的 System Prompts 和 JSON Schema
    └── logger.py            # 日志记录器（将 AI 的内部 OS 和完整对局保存到本地文件，方便复盘查错）
```
赛博圆桌会议、电子斗蛐蛐之八个ai一起玩阿瓦隆。

###目前存在的问题:
1.ai总是陷入毫无进展的吵架，无法正常推进第一轮游戏，纠结于第一轮的人选。

2.经常出现多个ai指责同一人的情况发生，这里强制判断有无重复指责才在表象上缓解了这个现象。

3.针对ai无休止复读机的情况，添加了讨论倒计时，以增加ai的紧迫感（但是反正我没看到ai有任何时间紧迫的感觉）

#####3.优化提示词
ai的性格是为了防止出现同质化发言，毕竟都调用的deepseek接口（调了半天千万token花五块钱 微信视频号一堆吵token翻译的，我感觉翻译成元炁的炁就好 bushi侠岚之魂熊熊燃烧 跟炼丹一样  没有炁就催动不了大模型），但是现在的ai过于表现自己的性格，哪怕是信息量最少的第一轮也难以选人上车。

需要想一套提示词，既能让ai有区分度，同时专注于游戏，能够以人类玩家的逻辑去玩，而不是聚焦于无用的选人。
